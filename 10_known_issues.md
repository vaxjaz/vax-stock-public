### ~~#D 主力净流入口径+单位双重错误~~ [已修 2026-06-19]
- **现象**: 报告"今日主力"恒瑞6/18显示≈0.03亿,与同花顺5.13亿、与涨跌+3%矛盾
- **根因**: tushare_source.py get_moneyflow_summary 用分档加总(与涨跌相关仅0.147的噪音)代替net_mf_amount(相关0.871),且单位×1000应为×10000
- **修复**: 改用 net_mf_amount×10000,与回测IC同源;实测恒瑞修复后+8.09亿

### ~~#E 融资买入比交易所不完整导致分位失真~~ [已修 2026-06-19]
- **现象**: 6/18融资比5.43%/3年第0%/✅,与6/17的10.72%/❌矛盾,误把宏观从🔴拉成🟡
- **根因**: margin接口某交易所当天未发布时,groupby.sum只聚合到部分交易所,分子腰斩
- **修复**: 交易所完整性校验(SSE+SZSE齐全),残缺丢弃+回退最近完整日+报告标注stale;历史缓存重拉清坏点(524→520,min 4.61%→6.81%)
- **教训**: 数据"未就绪"≠"缺失";聚合前必须校验来源完整性
## 🟠 待修(P1/P2,影响功能但有 workaround)

### #5 VPS cron 日志未实现 logrotate
- **现象**: cron.log 文件可能持续增长
- **影响**: 长期可能占满磁盘
- **Workaround**: stock_report_enhanced.py 末尾的 cleanup_old_files 会裁剪到 1000 行,但前提是脚本能跑完
- **预期修复**: 加 logrotate 系统配置,或改用 RotatingFileHandler
- **优先级**: P2
- **发现日期**: 2026-06-13
### #6 A500ETF 历史不足 3 年
- **现象**: A500ETF (159338.SZ) 2024-09 才成立,3 年百分位算不出
- **影响**: 维度1 ETF 净申赎在初始几个月可能不准
- **Workaround**: 代码已用 `min_periods=window//2` 兼容较短历史
- **预期修复**: 时间会自然解决,2027-09 之后即满 3 年
- **优先级**: P3
- **发现日期**: 2026-06-13
### #7 yc_cb 接口需 5000+ 积分
- **现象**: 维度4 ERP 的中债国债收益率接口 Vax 的 2000 积分账户无权限
- **影响**: 国债收益率无法自动更新
- **Workaround**: secrets.json 配 `yield_10y_pct`(默认 2.30%),每月手动从东方财富/中国债券信息网更新一次
- **预期修复**: 等 Tushare 积分升级到 5000+,或自动从公开 API 拉取
- **优先级**: P2
- **发现日期**: 2026-06-14
### #8 维度5 全A遍历首次回填耗时长
- **现象**: 全A 4826 只股票 × 250 天日K,首次回填约 53 分钟
- **影响**: 首次部署慢
- **Workaround**: 只跑一次,后续每日增量约 1-3 分钟
- **预期修复**: 可考虑用 multiprocessing 并发拉取(改造成本较高,优先级低)
- **优先级**: P3
- **发现日期**: 2026-06-14

## ✅ 已修复(归档区)

### ~~#1 ETF 净申赎单位错误 1万倍~~ [已修 2026-06-14]
- **现象**: 报告显示 5d=501318亿,真实值应该是 ~50亿
- **根因**: Tushare `fund_share.fd_share` 单位是万份(代码假设亿份),缺少 /10000 换算
- **修复**: `_compute_etf_net_sub` 函数加 `/ 10000`
### ~~#2 维度3 换手率指数代码错误~~ [已修 2026-06-14]
- **现象**: `index_dailybasic(ts_code="000985.CSI")` 返回空
- **根因**: Tushare 的 `index_dailybasic` 不支持 .CSI 后缀指数
- **修复**: 改为候选列表 `["000985.SH", "000300.SH"]`,逐个尝试
### ~~#3 维度4 ERP yc_cb 无权限~~ [已修 2026-06-14]
- **现象**: "抱歉,您没有接口(yc_cb)访问权限"
- **根因**: Tushare 2000 积分账户不能用 yc_cb
- **修复**: yc_cb 失败时 fallback 到 secrets.json 的 yield_10y_pct(默认 2.30%)
- **关联**: 升级为长期问题 #7(用户需每月手动维护)
### ~~#4 summary errors 字段失败时不上报~~ [已修 2026-06-14]
- **现象**: 维度3/4 失败,但 result["errors"] 是空数组
- **根因**: 内部 fetch 返回 None 时,外层 if 不进入,也不进 except
- **修复**: 6 个维度全部加 else 分支显式 append errors
### ~~#A portfolio.json 持仓字段缺 name~~ [已修 2026-06-13,通过 v2 架构]
- **现象**: holdings 里 `{600276: {"cost":..., "shares":...}}` 缺 name
- **根因**: 原 portfolio.json 设计疏漏
- **修复**: v2 架构后,holdings.json 强制要求 name 字段
### ~~#B tushare_token 明文存在 portfolio.json~~ [已修 2026-06-13]
- **现象**: token 上传到 Claude project 后会泄露
- **根因**: portfolio.json 把敏感配置和业务数据混在一起
- **修复**: 拆出 secrets.json(chmod 600,仅 VPS),portfolio.json 已废弃
### ~~#C HOLDINGS 全局变量在 v2 架构后变空~~ [已修 2026-06-13,by design]
- **现象**: stock_report_enhanced.py 里 HOLDINGS = {} 永远为空
- **根因**: v2 架构后 VPS 不管持仓
- **修复**: 这是 by design 不是 bug。HOLDINGS 保留是为了兼容 L2387/2406/2435 等下游代码不需要改
