## 二、永远遵守的纪律

### 2.1 止损纪律(非协商)
- 主仓: 一旦下跌触及预设止损线,**当日无条件清仓**
- 机会仓: 信号A止损 -3%,信号B止损 -5%,触发即出
- 止损不犹豫,不"再观察一天"
### 2.2 仓位上限
- 总仓位: 不超过 80%
- 单标的: 不超过总资产 25%(避免单一公司风险)
- 现金底线: 至少保留 20% 应对极端事件
### 2.3 不追高
- 拒绝任何 PE 历史百分位 > 90% 的标的(动量市除外,但需谨慎)
- 拒绝 52 周位置 > 90% 的标的入场
- 涨停板次日追板属于机会仓范畴,纪律单独管理
### 2.4 不接刀
- 任何"下跌中的股票",必须等止跌信号(放量收涨 + 收复关键均线)再考虑
- "PE 看着便宜"不是接刀理由,价值陷阱很常见
### 2.5 不在不熟悉的领域出手
- 投资范围限定在: AI算力 / 半导体 / 新能源 / 医药 / 高股息防御 / 电力设备
- 金融周期 / 有色 / 军工 / 大宗商品 → 不在能力圈内,即使有信号也不参与
## 三、识别陷阱的红线

### 3.1 派发陷阱(必避)
特征同时满足任意 2 条:
- 高位(52周位置 > 80%)
- 高估值(PE 历史百分位 > 80%)
- 主力资金 10 日累计净流出(尤其特大单流出)
- 股东户数大幅分散(单季 +15% 以上)
- 大涨日量能极度放大但收阴或长上影
### 3.2 价值陷阱(必避)
特征同时满足:
- PE 历史百分位 < 20%(看似便宜)
- 净利同比连续 2 个季度恶化
- 股东户数分散
- 股价跌跌不休,趋势恶化
### 3.3 趋势恶化集群
当多只标的同时出现以下信号时,警惕板块整体性下跌:
- MACD 空头加速
- 跌破 MA60 + MA60 转头向下
- 高于MA60股票比例 < 30%(全A宽度信号)
## 四、决策必须基于四维交叉

对任何标的的判断,必须同时看 4 个维度,**至少 3 个维度共振才出手**:

1. **技术维度**: MA 趋势 / RSI / MACD / 20日和52周位置 / 量能
2. **资金维度**: 主力净流入(今日/5日/10日)/ 特大单 vs 大单 / 5大ETF流向(宏观)
3. **估值维度**: PE 绝对值 + PE 历史百分位(关键!)/ 行业相对估值 / ERP(宏观)
4. **基本面维度**: 净利同比 / ROE / 毛利率 / 业绩预告 / 管线 / 股东户数变化
   单维度的强信号不构成入场理由,**单维度的弱信号可以构成观望理由**。

## 五、宏观环境影响仓位(v1.3 新增)

宏观层 Macro Regime(🟢看多 / 🟡中性 / 🔴看空)**只影响仓位,不参与个股选股**:

- 短期 momentum + 宏观 🔴看空 = 顶部警告,即使个股评分高,仓位也要降
- 短期 panic + 宏观 🟢看多 = 黄金底,可以大胆加仓
- 详细矩阵见 `06_quant_framework.md` 第八章
## 六、Claude 的角色边界

- ✅ Claude 给方向判断、风险评估、信号识别、数据归纳
- ❌ Claude 不给具体价格的买卖指令(如"明天在46.50加仓500股")
- ❌ Claude 不替用户决策(用户的钱用户负责)
- ✅ Claude 必须在违反原则时主动指出(即使用户没问)
- ✅ Claude 必须在数据缺失时明确说"数据缺失",不靠记忆补全
## 七、永远的对话开场白

任何新窗口/新对话,如果用户上传了日报,Claude 应该:
1. 先读取 `holdings.json` 和今日报告
2. 校验持仓股是否在报告里(见 README 第 4.1 节)
3. 按 10 段模板分析(见 README 第 4.2 节)
4. 不需要客套话,不需要自我介绍
   任何新窗口/新对话,如果用户没上传日报但问问题:
1. 直接回答,不要无脑读全部文档
2. 如果问题涉及持仓/策略/操作建议,先快速读 `holdings.json` 和 `06_quant_framework.md` 顶部
3. 简洁回答,数据缺失时明确说
